tipo de ataque Tarea Descripción de la Prueba
Fuerza Bruta Test: Bloqueo de Cuentas por Fallo Validar: El sistema bloquea al usuario o introduce CAPTCHA después de N intentos fallidos (e.g., 5 veces) en un periodo de tiempo.
Fuerza Bruta Test: Contraseñas Débiles/Comunes Verificar: El sistema rechaza contraseñas incluidas en listas negras o muy comunes.
Inyección SQL Test: Inyección en Campos (SQLi) Probar: Ingresar caracteres especiales (', --, OR 1=1) en los campos de usuario y contraseña para ver si se produce un error de base de datos o acceso no autorizado.
Ataque de Tiempos Test: Mensajes de Error Genéricos Asegurar: El mensaje de error es siempre el mismo ("Credenciales Inválidas") tanto para un usuario inexistente como para una contraseña incorrecta.
Ataque de Tiempos Test: Detección de Usuario (Enumeración) Evaluar: No debe haber diferencia notable en el tiempo de respuesta o en los mensajes de error que permita adivinar si un nombre de usuario existe o no.
Sesión Test: Regeneración de ID de Sesión Verificar: Después de un inicio de sesión exitoso, el ID de sesión del usuario debe cambiar.
Sesión Test: Inactividad de Sesión (Timeout) Validar: La sesión de un usuario expira automáticamente después de un período de inactividad (e.g., 15 minutos) y requiere volver a autenticar.
Hashing Test: Almacenamiento Seguro de Passwords Confirmar: Las contraseñas se almacenan con algoritmos de hashing fuerte y salted (como Argon2 o bcrypt), no en texto plano ni con MD5.
Seguridad de Capa Test: Uso de Autenticación de Dos Factores (2FA) Verificar: La opción de 2FA está disponible y funciona correctamente como una segunda capa de seguridad.
Seguridad de Capa Test: Uso de HTTPS/SSL Asegurar: Todas las comunicaciones del login utilizan cifrado HTTPS para proteger las credenciales en tránsito.
